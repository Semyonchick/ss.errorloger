# ss.errorloger

Bitrix-модуль для быстрого анализа PHP-ошибок: он читает штатную настройку
`exception_handling`, объединяет одинаковые ошибки, показывает число повторений,
первое и последнее срабатывание, файл, строку, хост и стек вызовов.

## Возможности

- источник журнала берётся из `Bitrix\\Main\\Config\\Configuration`;
- чтение последних 5 МБ без выполнения содержимого журнала;
- группировка по уровню, типу, сообщению, файлу и строке;
- поиск и компактная карточка каждой группы;
- одна отзывная ссылка без авторизации сроком ровно 72 часа;
- в Bitrix хранится только SHA-256 токена, ссылка закрыта от индексации;
- при отсутствии настройки на странице показывается готовый пример
  `.settings_extra.php`.

## Установка вручную

Разместите репозиторий в `local/modules/ss.errorloger`, затем установите модуль
«Анализ ошибок PHP» в административной части Bitrix.

## Установка через Composer

Пакет имеет тип `bitrix-d7-module`. В корневом `composer.json` проекта:

```json
{
  "repositories": [
    {
      "type": "vcs",
      "url": "https://github.com/Semyonchick/ss.errorloger.git"
    }
  ],
  "require": {
    "composer/installers": "^2.3",
    "semyonchick/ss.errorloger": "^1.0"
  },
  "config": {
    "allow-plugins": {
      "composer/installers": true
    }
  },
  "extra": {
    "installer-paths": {
      "local/modules/{$name}/": ["type:bitrix-d7-module"]
    }
  }
}
```

После `composer install` файлы будут находиться в
`local/modules/ss.errorloger`. Composer доставляет код, а установка/обновление
модуля в Bitrix выполняется отдельно — вручную или миграцией Sprint Migration.

## Источник журнала

Модуль ищет путь в `exception_handling.log.settings.file`. Пример для
`/bitrix/.settings_extra.php` показан прямо на странице модуля, если настройка
отсутствует. Существующую секцию `exception_handling` нужно дополнить, а не
дублировать.

## Временная ссылка

Администратор создаёт или отзывает ссылку на странице «Ошибки PHP». Токен
показывается один раз, действует 72 часа и заменяет ранее выданный токен.
