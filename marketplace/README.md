# Подготовка публикации в Bitrix Marketplace

## Метаданные решения

- код модуля: `ss.errorloger`;
- тип решения: модуль;
- название: «Анализ ошибок PHP»;
- партнёр: «Рецепты Рекламы» (`https://rere-design.ru`);
- разработчик: Семен Сушенцев (Semyonchick);
- лицензия: MIT;
- рекомендуемая модель: бесплатное решение;
- минимальная версия PHP: 7.4.

Перед созданием карточки решения убедитесь, что в партнёрском кабинете за вами
зарегистрирован код партнёра `ss`. Код карточки и `MODULE_ID` должны совпадать.

## Полная сборка

```powershell
pwsh tools/build-marketplace.ps1
```

Команда создаёт `dist/.last_version.zip` с каталогом `.last_version` внутри.
Архив содержит только runtime-файлы модуля и подходит для поля полной сборки.

## Чек-лист перед отправкой

- установить и удалить модуль на чистых UTF-8 и CP1251-установках Bitrix;
- проверить PHP 7.4, 8.1 и актуальную поддерживаемую версию PHP;
- включить вывод предупреждений Bitrix и убедиться, что установка и страницы их не создают;
- проверить административную страницу и временную ссылку в поддерживаемых браузерах;
- убедиться, что временная ссылка истекает через 72 часа и немедленно перестаёт работать после отзыва;
- проверить журнал с отсутствующим файлом, пустым файлом и файлом больше 5 МБ;
- подготовить логотип и скриншоты для карточки решения;
- загрузить сначала альфа-версию на тестовый лицензионный ключ партнёра.

## Ограничения

Репозиторий и архив подготовлены технически. Финальная отправка невозможна без
доступа к партнёрскому кабинету, подтверждённого кода партнёра `ss`, логотипа,
скриншотов и выбора поддерживаемых редакций продукта.
