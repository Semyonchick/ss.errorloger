# Безопасность

Не публикуйте уязвимости и примеры журналов с персональными данными в Issues.
Сообщите о проблеме через приватный GitHub Security Advisory репозитория.

Поддерживаются последняя стабильная версия и ветка `main`. Временная публичная
ссылка рассматривается как секрет: передавайте её только доверенному получателю
и отзывайте сразу после диагностики.
